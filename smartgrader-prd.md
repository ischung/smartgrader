## 1. 프로젝트 개요

### 1.1 Core Goal
교수가 여러 과목·분반의 성적을 하나의 시스템에서 일괄 관리하고,
과목별 유연한 성적 항목·가중치 설정으로 총점과 학점을 정확하게
자동 산출할 수 있게 한다.

### 1.2 배경 및 목적
- **대상 사용자**: 대학교수(관리자), 학생
- **사용자 Pain Point**: 과목마다 성적 항목과 가중치가 달라 총점·학점을 수작업으로 계산하는 번거로움; 학생은 본인 성적을 교수에게 직접 묻기 부담스럽고 공식 발표 전 확인 수단이 없음
- **기대 효과**: 엑셀 파일 업로드만으로 가중치 기반 총점·학점이 자동 계산되고, 학생도 본인 성적을 언제든지 직접 조회 가능

### 1.3 범위 (Scope)

**In-Scope (v1 포함)**
- 교수/관리자 계정 로그인 및 비밀번호 변경
- 교수 계정 등록·조회·수정·삭제 (관리자 전용)
- 과목·분반별 성적 파일(엑셀/CSV) 업로드·조회·삭제
- 성적 파일 업로드 시 과목 정보(과목명·교과코드·분반·학기) 자동 추출 및 수동 입력
- 성적 항목 관리 (이름 추가·수정·삭제, 항목 수 유동적)
- 가중치 설정 (% 단위, 합계 100% 검증)
- 결석시수 기반 출석 점수 자동 계산
- 태도 점수 기본값 초기화 및 수정
- 총점 자동 계산 (가중치 반영, 총점 열 생성)
- 학점 범위 설정 및 학점 자동 산출 (A+~F, 학점 열 생성)
- 성적 항목 점수 직접 수정·저장
- 결과 파일 엑셀 다운로드 (원본 파일 보존, 결과 파일 별도 저장)
- 학생 계정 자동 생성 (성적 파일 기반, 초기 ID·PW = 학번)
- 학생 로그인 후 수강한 모든 과목 성적 조회 및 비밀번호 변경

**Out-of-Scope (v1 제외)**
- 이메일 알림 / 성적 이의신청 기능
- 학생 성적 통계·분포 시각화 차트
- LMS 연동
- 모바일 앱 / 반응형 웹

## 2. 대상 사용자

**Primary Persona A — 교수(관리자)**
김인상, 대학교수. 매 학기 5~10개 과목·분반의 성적을 엑셀로 관리하며,
항목별 가중치가 과목마다 달라 매번 수작업으로 계산하는 데 시간을 낭비하고 있다.

**Primary Persona B — 학생**
이수강, 대학생. 성적이 어떻게 계산됐는지 알고 싶지만 교수에게 직접
묻기 부담스럽고, 공식 발표 전까지 본인 점수를 확인할 방법이 없다.

**핵심 니즈**
- 교수: 반복 수작업 제거, 정확한 계산, 과목별 유연한 설정
- 학생: 본인 성적의 투명한 조회

## 5. 상세 기능 요건

### 5.1 계정 관리

**Logic**:
1. 관리자(ID: 110509 / 초기 PW: insang)가 최초 로그인 후 비밀번호 변경 가능
2. 관리자가 교수 계정을 등록·조회·수정·삭제
3. 교수가 로그인하여 과목 성적 관리 기능 사용
4. 학생은 성적 파일 업로드 시 학번으로 계정 자동 생성 (초기 PW = 학번)
5. 학생이 첫 로그인 후 비밀번호 변경 가능

**Validation**:
- 교수 계정 등록 시 필수 입력: 이름, ID, 초기 비밀번호
- 학번은 중복 불가

**Edge Case**:
- 데이터 없음: 등록된 교수가 없으면 "등록된 교수가 없습니다. 교수를 추가해주세요." 안내
- 오류 발생: 로그인 실패 시 "앗, 아이디 또는 비밀번호가 올바르지 않아요." 메시지 표시

---

### 5.2 성적 파일 등록 및 관리

**Logic**:
1. 교수가 성적 파일(.xlsx, .xls, .csv) 업로드
2. 시스템이 파일에서 과목명·교과코드·분반·학기 정보 자동 추출 시도
   - 추출 성공 시: 추출된 정보를 교수에게 확인 요청
   - 추출 실패 시 (정보 없음): 교수에게 직접 입력 요청
     - 과목명 (필수)
     - 교과코드 (필수)
     - 분반 (선택, 없으면 단일 분반으로 처리)
     - 학기 (필수, 형식: YYYY-1 또는 YYYY-2, 예: 2026-1)
3. 동일 교과코드라도 분반이 다르면 별개의 과목으로 처리·조회
4. 과목 정보는 교수의 담당 과목 목록에 자동 등록
5. 업로드된 성적 데이터를 화면에서 미리보기로 확인
6. 성적 항목 이름 추가·수정·삭제 (중간점수, 기말점수, 과제1, 수시시험1 등)
7. 각 항목에 가중치(%) 설정 → 합계 100% 실시간 검증
8. 결석시수 항목 존재 시: 출석점수 = 출석가중치 만점 − (결석시수 × 차감점)으로 자동 계산
   - 결석시수 1회당 차감점은 교수가 직접 설정
9. 태도 점수 항목 존재 시: 가중치 만점으로 초기화, 교수가 개별 수정 가능
10. 개별 점수 직접 수정·저장 가능

**Validation**:
- 업로드 파일 형식: .xlsx, .xls, .csv 허용
- 가중치 합계가 100%가 아니면 총점 계산 버튼 비활성화 + "앗, 가중치 합계가 100%여야 해요!" 경고
- 학기 형식: YYYY-1 또는 YYYY-2만 허용

**Edge Case**:
- 데이터 없음: 등록된 성적 파일이 없으면 "아직 등록된 성적 파일이 없어요. 파일을 업로드해볼까요?" 안내
- 오류 발생: 파일 형식 불일치 또는 파싱 오류 시 "앗, 파일을 읽을 수 없어요. 엑셀 형식을 확인해볼까요?" 메시지 표시

---

### 5.3 총점 계산 및 학점 산출

**Logic**:
1. 교수가 가중치 설정 완료 후 "총점 계산" 실행
2. 각 학생의 총점 = Σ(항목 원점수 × 항목 가중치%)로 계산, 총점 열 자동 생성
3. 교수가 학점 범위 설정 (예: 95이상 → A+, 90~94 → A0 … 60미만 → F)
4. "학점 계산" 실행 시 총점 기준으로 학점 열 자동 생성 (A+, A0, B+, B0, C+, C0, D+, D0, F)
5. 결과 파일 = 원본 파일의 모든 항목(열) + 총점 열 + 학점 열
6. 원본 파일은 변경하지 않고 보존; 결과 파일은 별도로 저장 및 다운로드 제공
7. UI에서 원본 파일과 결과 파일을 명확히 구분하여 표시

**Validation**:
- 총점 계산은 가중치 합계 100% 검증 통과 후에만 실행 가능
- 학점 범위는 0~100 사이여야 하며 구간이 겹치거나 비어있으면 경고
- 학점 범위 설정은 과목별로 독립적으로 저장

**Edge Case**:
- 데이터 없음: 총점 계산 전 학점 계산 시도 시 "앗, 먼저 총점을 계산해야 해요!" 안내
- 오류 발생: 일부 학생의 점수 누락 시 "앗, 점수가 입력되지 않은 항목이 있어요. 확인 후 다시 계산해볼까요?" 경고

---

### 5.4 학생 성적 조회

**Logic**:
1. 학생이 학번/비밀번호로 로그인
2. 본인이 수강한 모든 과목 목록 표시 (학기별 구분)
3. 과목 선택 시 해당 과목의 성적 항목별 점수, 총점, 학점 조회
4. 다른 학생의 성적은 접근 불가

**Validation**:
- 학생은 본인 학번에 해당하는 데이터만 조회 가능
- 총점·학점이 아직 계산되지 않은 경우 해당 항목은 "미산출"로 표시

**Edge Case**:
- 데이터 없음: 등록된 성적이 없으면 "아직 등록된 성적 정보가 없어요." 안내
- 오류 발생: 로그인 실패 시 "앗, 아이디 또는 비밀번호가 올바르지 않아요." 메시지 표시

## 6. UI/UX 요건

- **레이아웃**: 대시보드형 — 로그인 후 주요 기능을 카드로 한눈에 표시
- **시각 스타일**: 직관적이고 아름다운 UI (교수·학생 모두 쉽게 사용 가능)
- **로딩 상태**: 스피너 (회전 아이콘)
- **오류 메시지 톤**: 친근하고 가볍게 ("앗, ~")

### 인터랙션

| 사용자 행동 | 시스템 반응 |
|------------|------------|
| 파일 업로드 | 스피너 표시 → 과목 정보 확인/입력 → 미리보기 화면으로 전환 |
| 가중치 입력 | 합계 실시간 표시 및 100% 초과/미달 시 즉시 경고 |
| 총점 계산 버튼 클릭 | 스피너 → 총점 열 추가된 결과 화면 표시 |
| 학점 계산 버튼 클릭 | 스피너 → 학점 열 추가된 결과 화면 표시 |
| 결과 파일 다운로드 | 즉시 .xlsx 파일 다운로드 |
